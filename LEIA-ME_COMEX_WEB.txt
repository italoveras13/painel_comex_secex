COMO GERAR O BANCO REDUZIDO PARA PUBLICAÇÃO

1. Primeiro, conclua o ETL completo. O arquivo abaixo deve existir:

   data\processed\comex.duckdb

2. Feche o Streamlit com Ctrl+C e, no PowerShell aberto na pasta do projeto,
   execute:

   .\.venv\Scripts\python.exe build_web_database.py

3. Aguarde a mensagem "Validação concluída". Com uma base de 5 GB, a geração
   pode demorar e precisa de espaço livre para o novo banco e arquivos
   temporários. Não feche a janela enquanto estiver processando.

4. O resultado será:

   data\processed\comex_web.duckdb

5. Confira o tamanho:

   Get-Item .\data\processed\comex_web.duckdb |
     Select-Object Name, @{Name="Tamanho_MB";Expression={[math]::Round($_.Length/1MB,2)}}

6. Inicie o site. Ele selecionará automaticamente a base web:

   .\.venv\Scripts\python.exe -m streamlit run app.py

7. Para gerar novamente depois de atualizar os CSVs:

   .\.venv\Scripts\python.exe build_web_database.py --force

O script informa ao final:
- até 100 MB: GitHub comum;
- acima de 100 MB e até 2 GB: Git LFS;
- acima de 2 GB: usar volume persistente ou aplicar nova redução.

O comex.duckdb completo não é alterado nem deve ser enviado ao GitHub.
